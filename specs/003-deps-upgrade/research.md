# Phase 0 Research: Whole-Repo Dependency Upgrade

## R-001: Vite 8 ↔ `@cloudflare/vite-plugin` Compatibility

- **Decision**: Bump `@cloudflare/vite-plugin` 1.25.6 → 1.33.2 alongside Vite 7 → 8.
- **Rationale**: 1.33.2 declares `peerDependencies.vite: "^6.1.0 || ^7.0.0 || ^8.0.0"` — first version to officially support Vite 8. It also requires `wrangler ^4.85.0`, which we satisfy via the wrangler bump.
- **Source**: `npm view @cloudflare/vite-plugin@1.33.2 peerDependencies` ⇒ `{ vite: '^6.1.0 || ^7.0.0 || ^8.0.0', wrangler: '^4.85.0' }`.
- **Alternatives considered**: Stay on Vite 7 + cloudflare-plugin 1.33. Rejected — defers the risk and forces a second upgrade window. We accept the risk now while the rest of the surface area is fresh in mind.

## R-002: `@vitejs/plugin-react` 6 Peer Requirements

- **Decision**: Bump `@vitejs/plugin-react` 5.1.4 → 6.0.1; do NOT install the optional Babel peers.
- **Rationale**: 6.0.1 declares `peerDependencies.vite: "^8.0.0"` (forces Vite 8) and two optional peers (`@rolldown/plugin-babel`, `babel-plugin-react-compiler`) flagged via `peerDependenciesMeta.optional`. We don't use the React compiler or rolldown.
- **Source**: `npm view @vitejs/plugin-react@6.0.1 peerDependencies` and `peerDependenciesMeta`.
- **Alternatives considered**: Hold plugin-react at 5. Rejected — drops support; bug fixes only land in 6.x.

## R-003: Vite 8 Node Engine

- **Decision**: Tighten `package.json#engines.node` from `"22.x"` to `">=22.12"`.
- **Rationale**: Vite 8.0.10 declares `engines.node: "^20.19.0 || >=22.12.0"`. Our floor was `22.x` (i.e., 22.0+), which silently violates the peer on a 22.0–22.11 contributor box. Tightening makes the requirement explicit at install time instead of at first build.
- **Source**: `npm view vite@8.0.10 engines`.
- **Alternatives considered**: Keep `22.x` and trust contributors. Rejected — fails-loud at `pnpm install` is better than fails-late at `vite build`.

## R-004: TanStack Start + Router Compat With Vite 8

- **Decision**: Bump `@tanstack/react-router` 1.163.2 → 1.168.24 and `@tanstack/react-start` 1.163.2 → 1.167.49.
- **Rationale**: react-start 1.167 declares `peerDependencies.vite: ">=7.0.0"`, which Vite 8 satisfies. Router has no Vite peer.
- **Source**: `npm view @tanstack/react-start@1.167.49 peerDependencies` ⇒ `{ vite: '>=7.0.0' }`.
- **Alternatives considered**: Pin TanStack at 1.163. Rejected — releases since 1.163 contain bug fixes for the Workers SSR path.

## R-005: TypeScript 6 Risk Surface

- **Decision**: Bump `typescript` 5.9.3 → 6.0.3; fix any code-level errors; do not weaken `tsconfig.json`.
- **Rationale**: TypeScript 6 has documented stricter inference for some patterns. The most likely failure points in this codebase are (a) Drizzle ORM 0.45 type defs, (b) `@tanstack/react-router` route generation, (c) `worker-configuration.d.ts` (regenerated by `pnpm cf-typegen`).
- **Source**: TypeScript 6.0 release notes; constitution Principle III (strictness must not be lowered).
- **Alternatives considered**: Skip TS 6. Rejected — user explicitly opted into the major bump.

## R-006: Vitest 4 + Vite 8 Plugin API

- **Decision**: Verify `vitest.config.ts` still loads `@vitejs/plugin-react@6` after the Vite 8 bump; no API changes anticipated.
- **Rationale**: Vitest 4.1.5 is already on latest; the project already uses `defineConfig` from `vitest/config`. The risk is plugin-react 6 (`vite: ^8.0.0` peer) failing to bind under Vitest's internal Vite. Verified empirically by running `pnpm test:coverage` after the bump.
- **Alternatives considered**: Pin plugin-react at 5 to avoid Vitest risk. Rejected — plugin-react 6 follows Vite 8 by definition.

## R-007: AWS SDK v3 Minor Jump (3.999 → 3.1037)

- **Decision**: Bump `@aws-sdk/client-s3` to latest minor (3.1037.x).
- **Rationale**: AWS releases up to multiple times per week; 38 minors of churn typically introduces no breaking changes at the SDK consumer level (`PutObjectCommand`, `DeleteObjectCommand`, `S3Client`). Verified by running the existing `services/cloudflare.ts` tests after the bump.
- **Alternatives considered**: Stay on 3.999. Rejected — security-relevant AWS SDK fixes accumulate.

## R-008: PR Body Skipped-Upgrades Discipline

- **Decision**: Always include a `## Skipped upgrades` section in the PR body. Empty case reads "(none)".
- **Rationale**: A silent skip becomes hidden tech debt. An auditable skip becomes a follow-up ticket.
- **Source**: spec.md FR-006.
- **Alternatives considered**: Omit when empty. Rejected — reviewers would have to scan the diff to confirm completeness.
